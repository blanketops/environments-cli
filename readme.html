platform · bootstrap · cli
blanketops-environments
Self-contained Kubernetes platform bootstrapper — no kubectl required. A single binary installs a complete cloud-native delivery stack directly against the cluster API.
Go Kubernetes client-go Gokrazy-compatible Static binary
✦
Platform stack
The installer deploys these components in dependency order, waiting for CRDs to settle between phases.
Kapp Controller packaging & lifecycle
Argo Events event-driven pipelines
Tekton Pipelines CI/CD execution
Tekton Dashboard pipeline UI
Shipwright Build native image builds
Crossplane infra orchestration
External Secrets Operator secure secret integration
Knative Serving serverless workload runtime
Kourier Knative ingress & networking
→
Installation order
Carvel→ Argo Events→ Tekton→ Dashboard→ Shipwright→ Crossplane→ Ext. Secrets→ Knative→ Kourier
⚙
Pure Go apply engine
All resources applied directly via the Kubernetes API. No external tooling required.
1
CRD detection — discover which custom resource definitions are needed
2
CRD installation — apply via client-go dynamic client
3
CRD registration wait — poll until the discovery REST mapper sees new types
4
Resource application — decode unstructured objects, apply remaining manifests
◈
Design goals
Built for environments where traditional tooling may not exist.
Air-gapped clusters Bare metal nodes Immutable appliances Edge deployments Gokrazy systems Ephemeral CI clusters
▣
Fully embedded assets
All manifests and scripts are compiled into the binary via go:embed. Nothing is read from disk at runtime.
dependencies/
  ├── carvel/       ↳ Kapp Controller manifests
  ├── argoevents/   ↳ Argo Events manifests
  ├── tekton/       ↳ Tekton + Dashboard
  ├── shipwright/   ↳ Shipwright Build
  ├── knative/      ↳ Serving CRDs + core
  └── kourier/      ↳ net-kourier
scripts/
  ├── install-crossplane.sh
  ├── install-externalsecrets.sh
  ├── setup-shipwright-cert.sh
  └── setup-knative-kourier.sh
🔐
Shipwright webhook cert automation
CSR generation
Creates a certificate signing request for the webhook endpoint
Certificate approval
Automatically approves the CSR via the Kubernetes certificates API
TLS secret creation
Stores the signed cert and key in a Kubernetes Secret
CA bundle injection
Patches the MutatingWebhookConfiguration with the CA bundle
No manual certificate management required. Webhook restart and deployment readiness checks are included.

Usage
$bops-env install
$bops-env uninstall
$bops-env self install
$bops-env self uninstall
$bops-env dependencies install
$bops-env dependencies uninstall
$bops-env cluster up [name]
$bops-env cluster down [name]
$bops-env cluster status [name]
⚒
Build & install
$mage build# standard binary
$mage static# fully static (Gokrazy-compatible)
$mage install# → $HOME/.local/bin or $HOME/bin
❄
Gokrazy workflow
$mage static
$gok add ./bin/bops-env-static
$gok build
🧪
Local testing
$kind create cluster
$bops-env install
$bops-env dependencies install
$kubectl get pods -A
